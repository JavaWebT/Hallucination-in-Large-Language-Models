### 出现幻觉的原因
- 数据缺陷： 数据质量低，过拟合；
- 训练缺陷
   - 预训练阶段：
     - 架构缺陷: 单向建模，上下文提取能力弱；自注意力，随着token长度增加，导致注意力稀释。
     - 暴露偏差：后面的token依赖前面生成的token,若前面的有错，则会出现级联错误。
   - 对齐阶段
     - 能力错位: LLM的内部能力与其被训练的标注信息不对应，出现知识边界，而产生幻觉。
     - 信念错位：基于人类反馈强化学习RLHF微调，导致LLMs预测会偏向人类的喜好，忽视事实性。
### 检测大模型病症
- 事实性幻觉
  - 检索外部事件： 预测结果与可靠的事实进行对比
  - 不确定性估计
    - 基于模型内部状态：主要依赖于访问大模型的内部状态。例如，通过考虑关键概念的最小标记概率来确定模型的不确定性。
    - 基于响应：则主要依赖于观察大模型的行为，不需要访问其内部状态。例如，通过采样多个响应并评估事实陈述的一致性来检测幻觉。
- 忠实性幻觉
  - 基于事实的度量，测量生成内容和源内容之间事实的重叠程度来评估忠实性。
  - 分类器度量：使用训练过的分类器来区分模型生成的忠实内容和幻觉内容。
  - 问答度量：使用问答系统来验证源内容和生成内容之间的信息一致性。
  - 不确定度估计：测量模型对其生成输出的置信度来评估忠实性。
  - 提示度量：让大模型作为评估者，通过特定的提示策略来评估生成内容的忠实性。

### 缓解幻觉的影响的方法
- 数据相关的幻觉
  - 高质量数据
  - 知识边界：
    - 知识编辑
    - RAG（检索增强生成）
    - 检索增强具体分为三种类型：一次性检索、迭代检索和事后检索。
      - 一次性检索是将从单次检索中获得的外部知识直接预置到大模型的提示中；
      - 迭代检索允许在整个生成过程中不断收集知识；
      - 事后检索是基于检索的修订来完善大模型输出。
      - <img width="1080" height="558" alt="image" src="https://github.com/user-attachments/assets/3186a4e5-eedf-40b2-ae02-e9ebd9bd194a" />
- 训练相关的幻觉
  - 过完善预训练策略
  - 确保更丰富的上下文理解
  - 和规避偏见来应对
- 推理相关的幻觉
  - 事实性增强解码
  - 忠实性增强解码
    - 上下文一致性
    - 逻辑一致性

# 当人工智能听起来正确却错误：理解幻觉、准确性与隐藏错误
<img width="1200" height="600" alt="image" src="https://github.com/user-attachments/assets/b19c87b6-fe24-4b3d-bc0a-f6f80060968a" />
像ChatGPT、Copilot和Claude这样的AI工具如今已深度融入我们的日常工作流程。我们依赖他们来写作、分析、编码，甚至支持决策。

但有一个许多人忽视的关键现实：人工智能并不总是准确的。更重要的是，它往往是自信地错误的。

如果不加审查，AI可能会在你的作品中引入细微但严重的错误。理解原因以及如何检测这一点至关重要。

### 人工智能不思考——它预测
从本质上说，人工智能不像人类那样思考或推理。它通过基于海量训练数据预测模式来生成响应。

这意味着它无法真正理解信息，不核实事实，也不可靠地识别错误。相反，它输出的声音听起来是正确的。

这就是为什么人工智能能够生成流畅且结构良好的回答，同时仍会犯下基本错误。
### 什么是人工智能幻觉？
人工智能幻觉是指模型生成的信息是虚假、误导或完全捏造的，但却将其呈现为事实。

挑战在于，幻觉反应乍看之下往往与正确的反应难以区分。

### 常见的幻觉类型
人工智能的幻觉通常以几种反复出现的模式出现：

虚构事实：
人工智能可能会生成看似可信的统计数据，比如“65%的团队在使用该工具后提高了生产力”，即使根本没有此类数据。
伪造引用：
可能引用的来源如“2023年哈佛研究”，这些来源要么无法验证，要么完全捏造。
混合现实错误：
人工智能可以结合真实和错误的信息，比如将虚假成就归因于真实人物，或描述真实公司的错误产品。
虚构实体：
在某些情况下，人工智能创造完全虚构的公司、事件或政策。

### 这些错误发生的原因
这些问题不是漏洞，而是AI工作方式固有的。

用模式填补空白：当人工智能缺乏信息时，它会基于相似模式生成反应，而非承认不确定性。
优化流畅度：AI被训练为听起来连贯自然，而非保证正确。
缺乏验证：没有外部系统，人工智能无法验证其生成的内容。

### 其他细微的AI错误
除了幻觉之外，还有一些更难被发现的问题。

偏见：AI反映了训练数据中存在的偏见，影响语言、推荐和框架。
模糊的回复：答案听起来合理，但缺乏可操作的见解。
上下文误解：人工智能可能误解意图，忽视细微差别，或未能连接相关输入。


### 真实案例：在实践中发现错误
想想这些问题在现实中是如何出现的。

示例1：商业统计
当被问及“有多少比例的远程工作者更有生产力？”时，人工智能可能会回答：“
根据2024年一项全球研究，78%的远程工作者更高效。”

这引发了担忧，因为过于精确的数字、缺乏可验证的来源以及过于自信的语气。

例子2：计算错误对于
“如果5个人每周节省3小时，那么每年节省多少？”这个问题，人工智能可能会回答：“
每年800小时。”

正确的计算是780小时，表明人工智能能够生成合理但错误的数字。

例子3：虚构的投资回报率指标
当被问及投资回报率时，人工智能可能会声称：“
公司通常在第一年内能看到4.2倍的投资回报率。”

这包括一个具体但没有任何假设或参考的图表。

例子4：近期事件
针对近期法规，AI可能会生成诸如“
首月有847家公司被罚款......”

此类回应往往包含高度具体的数字，缺乏可信来源，尤其是针对近期事件。

例子5：语境误解
像“写一封礼貌的邮件，拒绝录用通知，同时保持关系积极”这样的提示，可能会得到技术上正确但情感上直白的回复，未能达到预期语气。

示例6：通用回答
例如“我们如何提升团队生产力？”的问题可能会得到：“
重要的是要考虑多个因素，采取整体方法......”

虽然这并非错误，但缺乏具体性和可操作价值。
